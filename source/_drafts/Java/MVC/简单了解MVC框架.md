## MVC简介
>- (模型)M-Model :模型:业务逻辑和业务数据的处理,包括实体类 DAO层 Service业务层
>- (视图)V-View :视图层 负责界面和用户交互,不包含业务逻辑的JSP
>- (控制器)C-Controller :控制层:模型层和视图层之间的桥梁  用于控制业务流程

## MVC的作用
>- 将M和V的实现代码分离，从而使同一个程序可以使用不同的表现形式
